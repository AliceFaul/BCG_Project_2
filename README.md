# **QUY TẮC**

* Mỗi cuối ngày phải push task lên Git mặc kệ xong hay không xong
* Mỗi thành viên làm trên branch có tên trùng với task của mình trong tasksheet
* Tạo pull request xong phải báo với leader để review và merged vào main
* Commit task sẽ đặt tên theo quy tắc sau:
1. add/... : Khi thêm một chức năng gì đó
2. bugfix/... : Khi sửa bug của chức năng gì đó
3. refactor/... : Khi cải tiến code
4. style/... : Chỉnh format, tên biến (Không ảnh hưởng đến logic của chức năng)
